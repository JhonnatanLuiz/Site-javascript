JavaScript Array Const

ECMAScript 2015 (ES6)

Em 2015, o JavaScript introduziu uma nova palavra-chave importante: const.

Tornou-se prática comum declarar arrays usando const:

const carros = ["Fiat", "Volkswagen", "BMW"];

Não Pode Ser Reatribuído

Um array declarado com const não pode ser reatribuído:

const carros = ["Fiat", "Volkswagen", "BMW"];

carros = ["Toyota", "Honda", "Audi"];  // ERRO!

Erro! Você não pode reatribuir um array const a outro array. Isso vai gerar um TypeError.

Arrays NÃO São Constantes

A palavra-chave const pode ser um pouco enganosa.

Ela NÃO define um array constante. Ela define uma referência constante a um array.

Por causa disso, ainda podemos mudar os elementos de um array const.

Entendendo: O const impede que a variável aponte para outro array, mas não impede alterações no conteúdo do array.

Elementos Podem Ser Alterados

Você pode mudar os elementos de um array constante:

// Você pode criar um array constante:
const carros = ["Fiat", "Volkswagen", "BMW"];

// Você pode mudar um elemento:
carros[0] = "Toyota";

// Você pode adicionar um elemento:
carros.push("Audi");

// carros agora é ["Toyota", "Volkswagen", "BMW", "Audi"]

Deve Ser Inicializado na Declaração

Variáveis const devem receber um valor quando são declaradas:

// ERRADO - Isso NÃO vai funcionar:
const carros;
carros = ["Fiat", "Volkswagen", "BMW"];

// CORRETO - Deve inicializar na declaração:
const carros = ["Fiat", "Volkswagen", "BMW"];

Arrays declarados com var podem ser inicializados a qualquer momento. Você pode até usar o array antes dele ser declarado (hoisting):

// Isso funciona com var (hoisting):
carros = ["Fiat", "Volkswagen", "BMW"];
var carros;

Const Block Scope

Um array declarado com const tem escopo de bloco.

Um array declarado em um bloco não é o mesmo que um array declarado fora do bloco:

const carros = ["Fiat", "Volkswagen", "BMW"];
// Aqui carros[0] é "Fiat"

{
    const carros = ["Toyota", "Honda", "Audi"];
    // Aqui carros[0] é "Toyota"
}

// Aqui carros[0] é "Fiat"

Um array declarado com var não tem escopo de bloco:

var carros = ["Fiat", "Volkswagen", "BMW"];
// Aqui carros[0] é "Fiat"

{
    var carros = ["Toyota", "Honda", "Audi"];
    // Aqui carros[0] é "Toyota"
}

// Aqui carros[0] é "Toyota" (modificou o original!)

Redeclarando Arrays

Redeclarar um array com var é permitido em qualquer lugar:

var carros = ["Volvo", "BMW"];     // Permitido
var carros = ["Toyota", "BMW"];   // Permitido
carros = ["Volvo", "Fiat"];       // Permitido

Redeclarar ou reatribuir um array const, no mesmo escopo ou bloco, não é permitido:

var carros = ["Volvo", "BMW"];     // Permitido
const carros = ["Volvo", "BMW"];  // NÃO Permitido

{
    var carros = ["Volvo", "BMW"];     // Permitido
    const carros = ["Volvo", "BMW"];   // NÃO Permitido
}

Redeclarar um const existente, no mesmo escopo ou bloco, não é permitido:

const carros = ["Volvo", "BMW"];   // Permitido
const carros = ["Volvo", "BMW"];   // NÃO Permitido
var carros = ["Volvo", "BMW"];     // NÃO Permitido
carros = ["Volvo", "BMW"];         // NÃO Permitido

Redeclarar um array com const em outro escopo ou bloco é permitido:

const carros = ["Volvo", "BMW"];   // Permitido

{
    const carros = ["Volvo", "BMW"]; // Permitido
}

{
    const carros = ["Volvo", "BMW"]; // Permitido
}

Tornando o Array Realmente Imutável

Se você quer que o array seja completamente imutável (sem poder alterar elementos), use Object.freeze():

const carros = ["Fiat", "Volkswagen", "BMW"];
Object.freeze(carros);

// Agora nenhuma alteração funciona:
carros[0] = "Toyota";    // Silenciosamente falha
carros.push("Audi");     // Erro em strict mode

console.log(carros);     // ["Fiat", "Volkswagen", "BMW"]

Nota: Object.freeze() é "shallow" (superficial). Se o array contém objetos, esses objetos ainda podem ser modificados. Para congelamento profundo, você precisa congelar cada objeto individualmente.

Resumo: var vs let vs const

Característica var let const
Block Scope ✗ ✓ ✓
Hoisting ✓ ✗ ✗
Pode reatribuir ✓ ✓ ✗
Pode redeclarar ✓ ✗ ✗
Deve inicializar ✗ ✗ ✓

Recomendação: Use const por padrão. Use let quando precisar reatribuir. Evite var no código moderno.
